Compensate 补偿逻辑

场景 补偿动作
通用 验证源文件存在且大小 ≥ CheckpointLSN。若文件被截断/删除则返回错误。
导入的特殊性：与导出不同，导入的"补偿"不是修改目标数据库（数据库通常不支持截断）。补偿由 Execute 的读取位置 + DataTarget 的幂等性 共同保证：

1. Execute 从 CurrentReadOffset 开始读取，自然跳过 Store 已确认的数据
2. 崩溃前未 checkpoint 的脏写入（目标库中已存在但 Store 未记录），由 DataTarget 的实现方 通过 UPSERT / 唯一键去重 / 预检来处理
3. 引擎本身不负责目标库的回滚 — 这被设计为 DataTarget 接口的实现契约

DataTarget 接口契约（重要）

type DataTarget interface {
    // WriteBatch 批量写入行。
    //
    // 契约：实现方应保证写入的幂等性。
    // 例如使用 INSERT ... ON CONFLICT DO NOTHING，
    // 或通过业务唯一键在写入前检查是否已存在。
    //
    // 这是必须的，因为进程崩溃恢复时，
    // 上一批已入库但未 checkpoint 的数据会被重新读取和写入。
    WriteBatch(ctx context.Context, rows []Row) (inserted int64, err error)
}

Progress 映射

计算方式
CurrentReadOffset * 100 / SourceFileSize — 按源文件读取进度线性映射 0-100%

CheckpointLSN 语义

导入场景中 CheckpointLSN = 源文件中已安全解析并写入的字节偏移量。

恢复时引擎从该偏移量继续读取，DataTarget 的幂等性保证重复写入不产生脏数据。
